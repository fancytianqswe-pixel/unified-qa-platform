## 1. 背景与产品定位

### 1.1 背景

企业在数据质量、报账影像、合同与业务库表治理等场景中，需要一套**以自然语言为入口**、可**接入数据源**、可**按规则批量质检**、并可**将成熟流程沉淀为可复用技能与定时任务**的平台。传统做法依赖多人协作的配置后台 + 分散脚本，门槛高、不可追溯、难以复用。

### 1.2 产品定位

**统一质检平台**是面向业务与数据团队的 **AI 原生质检工作台**：

- **对话即入口**：用户用自然语言描述目标（接入库表、跑规则、解析文档、生成报告等）。
- **过程可观测**：执行过程可展开查看工具调用与中间结果，而非黑盒等待。
- **结果可沉淀**：认可流程可注册为技能，进入技能中心管理与复用。
- **任务可运营**：长耗时与定时类工作进入任务中心统一查看、重试与调度。

### 1.3 Demo 与生产的关系说明

当前仓库为 **Demo / 联调版本**，用于验证体验与集成路径。本 PRD 按 **生产级平台** 描述能力边界与验收标准；研发实现时可选用开源 Agent 网关（如 Hermes Agent）或同类能力，**不要求**与 Demo 目录结构一一对应。

## 2. 产品目标

### 2.1 业务目标

| 编号 | 目标 | 可衡量结果（生产建议） |
|------|------|------------------------|
| G1 | 降低数据源接入配置成本 | 业务用户可在 10 分钟内完成「库+表」级接入并连通性通过 |
| G2 | 提升规则质检效率 | 单次质检任务可输出通过率、异常样例、可导出报告 |
| G3 | 沉淀可复用质检流程 | 对话产出的流程可一键/确认后注册为技能，7 日内复用率可统计 |
| G4 | 支撑定时与批处理 | 技能可配置 Cron/间隔调度，任务中心可查看最近 30 天执行记录 |
| G5 | 满足企业基本安全合规 | 凭据不明文展示、**登录/操作/对话可审计**、权限可配置到菜单与按钮级 |
| G6 | 技能可迭代、可版本化、可授权发布 | 测试任务验证后转正式任务，技能可持续优化 |
| G7 | 支撑企业级接入 | 对接浙江电信 4A、DCOOS 等开放平台（生产） |

### 2.2 本期范围（MVP+，面向可上线 Demo 扩展）

**【已实现·Demo】包含：**

- 账号登录、组织/用户/角色与菜单按钮权限。
- 新任务（对话工作台）：多轮对话、附件、技能卡片、模型选择、内置助手入口。
- 技能中心（基础）：内置技能 + 用户技能列表/详情/新建/注册。
- 数据中心：结构化数据源（库表）与文件类数据源管理、连通性测试、字段选择与样例预览。
- 任务中心（基础）：任务列表、状态、来源、错误摘要（**无测试/正式任务分型、无逐单详情与标注**）。
- 系统管理：常规（语言/主题/退出）、用户与权限、MCP 服务、模型配置。
- 历史会话：侧栏历史进入 `/conversation/{id}` 只读或续聊（生产需明确策略）。

**【V1.1·规划】新增（本 PRD 重点增量）：**

- **系统管理 → 日志中心**：登录日志、操作日志、对话日志（查询、导出、权限控制）。
- **任务中心（增强）**：测试任务 / 正式任务双轨；逐单数据列表与详情；人工标注；任务级执行日志；AI 分析优化 Skill；测试任务转正式任务。
- **技能中心（增强）**：版本管理、自我迭代策略、市场授权与审核、评论打分、扩展内置技能（含「优化技能」等）。
- **非功能**：对话流畅度与自愈、百用户千对话并发、电信 4A/DCOOS 对接。

**暂不包含（生产规划单列，本期不验收）：**

- 完整 OA/审批流、工单闭环。
- 可视化 DAG 编排器。
- 全量多租户计费与配额中心。
- 独立「报告中心」独立产品模块（任务详情内嵌报告即可）。

### 2.3 非目标（明确不做）

- 替代企业统一身份源（生产应对接 LDAP/SSO，平台只做会话与权限映射）。
- 替代专业 ETL 平台做全量数据同步。
- 在平台内实现 MinerU/ OCR 算法的自研（应通过 **文档解析服务 + MCP/工具** 集成）。

## 3. 用户与角色

### 3.1 角色定义

| 角色 | 典型用户 | 核心诉求 |
|------|----------|----------|
| 超级管理员 | 平台运维/研发负责人 | 用户与权限、模型与 MCP、全局策略 |
| 质检业务用户 | 业务分析、财务质检 | 对话完成任务、查看报告、复用技能 |
| 数据管理员 | DBA、数据治理 | 维护数据源、字段映射、连通性 |
| 技能运营 | 业务专家 | 维护技能包、发布到「市场」、试跑 |
| 只读审计 | 内控 | 查看任务与日志，不可改配置 |

### 3.2 权限模型（产品规则）

- 权限粒度：**菜单（页面）** + **页面内按钮** 两级。
- 侧栏菜单：新任务、技能中心、任务中心、数据中心、历史任务（会话列表）、系统管理（弹层）。
- 系统管理子菜单：常规、用户与权限、MCP 服务、模型配置、**日志中心**（【V1.1·规划】）。
- 任务中心内：**任务执行日志**入口归属任务详情，不放入日志中心（【V1.1·规划】）。
- 无权限时：隐藏入口；直链访问时重定向到首个有权限页面或 403 页。
- 生产建议：权限数据存 **平台库**，支持从外部 IAM 同步用户，角色映射本地维护。

### 6.3 过程区交互规范（产品规则）

- 过程步骤：**无序号**、**无单步「完成」角标**；摘要为浅灰胶囊，技术细节默认收起。
- 工具步摘要应为**意图句**（如「向解析服务提交异步任务」），非整段 JSON。
- 仅有工具步的回合结束后，展示「已完成 + 耗时」折叠头；纯文本问答可不展示该折叠头。

（交互细则可对照设计稿《LLM对话UI规范》，研发实现保持一致即可。）

## 7. 模块二：技能中心

### 7.1 模块概述

**路由：** `/skills-center`、`/skills-center/[id]`、`/skills-center/new`  
**价值：** 管理内置与用户技能，支撑对话固化、市场展示、版本迭代与授权发布；与任务中心「AI 分析」共用优化能力。

**成熟度：** 列表/详情/新建/注册为 **【已实现·Demo】**；版本、自我迭代、授权审核、评论打分为 **【V1.1·规划】**。

### 7.2 功能需求 — 基础（【已实现·Demo】）

| 需求 ID | 描述 | 优先级 |
|---------|------|--------|
| SK-01 | 列表：搜索、分类（市场精选/我的）、内置技能标识 | P0 |
| SK-02 | 内置技能至少包含：**创建技能**、**接入数据源**、**数据质检（规则审核）**、**优化技能**（见 §7.4） | P0 |
| SK-03 | 详情：名称、描述、类型、SKILL 文档预览/源码、订阅状态（若启用） | P0 |

## 12. 内置场景与智能体能力（产品能力包）

## 14. 验收标准（摘要）

### 14.0 【V1.1·规划】增量验收（Phase 2+）

1. **日志中心**：管理员可检索登录/操作/对话日志；普通用户不可见他人对话正文。  
2. **测试任务闭环**：对话试跑 → 口令创建测试版 Skill → 口令跑批 → 任务列表可见 → 单条重试 → 人工标注 → AI 分析 → 一键应用 Skill 新版本。  
3. **正式任务**：测试任务转正式 → Cron 执行 → 按批次/日期 AI 分析 → Skill 版本更新可追溯。  
4. **技能中心**：版本历史、开放审核、市场评论；自我迭代策略可配置且产生版本日志。  
5. **并发压测**：100 用户、1000 并发对话连接下 P95 首 token 与错误率达标（指标见 §13.2）。  
6. **4A/DCOOS**：SSO 登录成功；至少 1 个能力经 DCOOS 注册并可被外部系统调用（试点）。

### 14.1 冒烟路径（必须通过）

1. 管理员登录 → 配置模型 → 测试连接成功。  
2. 业务用户登录 → 新任务 → 数据源向导 → 草稿卡保存 → 数据中心可见。  
3. 新任务 → 规则审核 → 产生过程工具条 → 完成报告 → 任务中心有记录。  
4. 固化技能 → 技能中心可见 → 新任务试用该技能。  
5. 普通用户无系统管理入口；权限矩阵只读生效。

### 14.2 体验性验收

- 长任务 ≥ 30s 时界面有「仍在执行」类反馈，而非永久「思考中」。  
- 过程区不出现「已完成工具仍显示准备参数进行中」的明显错乱。  
- 切换账号后看不到他人会话标题。
