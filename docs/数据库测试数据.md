## 日常启动（未注册 Windows 服务时）

机器重启或手动结束 `mysqld` 后，需再次启动进程（否则 `127.0.0.1:3307` 无监听）。在 **PowerShell** 中执行（路径与 ini 按本机实际修改）：

```powershell
Start-Process -FilePath "C:\Program Files\MySQL\MySQL Server 8.4\bin\mysqld.exe" `
  -ArgumentList "--defaults-file=$env:USERPROFILE\mysql-test.ini" `
  -WindowStyle Hidden
```

可用 `netstat -ano | findstr :3307` 或 `mysql -h 127.0.0.1 -P 3307 -u root -p` 验证。需要开机自启请见下文「注册 Windows 服务」。

### 报账单导入逻辑（需求说明）

1. **建表**：脚本启动时读取 `docs/qa_test_baodan_schema.sql`（UTF-8），通过 `mysql2` 一次执行（`multipleStatements`），避免在 Windows 下用管道把 SQL 喂给 `mysql.exe` 时中文列名被损坏。设置 `SKIP_SCHEMA=1` 可跳过（表已存在且结构一致时）。
2. **数据来源**：`scripts/import_qa_test_baodan.mjs` 内 `REL_FILES` 列出的 JSON 路径（相对仓库根）。
3. **字段映射**：`doc.基本信息` 中键名与表列名一致，按 `BASIC_KEYS` 顺序写入；`doc.合同履约信息.toatlpayment` 写入列 **`合同履约信息_toatlpayment`**（数值去逗号后 `DECIMAL`）。
4. **对象存储声明列（MinIO 等）**：表列 **`附件存储类型`**、**`附件存储桶`**、**`附件对象前缀`**。导入时若环境变量已设置 **`MINIO_BUCKET`**，则写入 `minio`、桶名、以及 **`{MINIO_OBJECT_ROOT}/<报账单号>/`**（`MINIO_OBJECT_ROOT` 默认 **`报账单`**，与 `scripts/sync_baodan_to_minio.mjs` 对象键一致）。未设置 `MINIO_BUCKET` 时三列为 NULL。**存量库加列**：执行 `docs/baodan_reimbursement_add_storage_columns.sql`。**仅补三列不写 JSON**：`MINIO_BUCKET=baodan-attachments node scripts/patch_baodan_minio_columns.mjs`。**删库中已弃用列**（若仍保留历史表结构）：`docs/baodan_reimbursement_drop_legacy_attach_columns.sql` 删除 **`本地附件目录`**、**`附件索引`**。
5. **MinIO 本机零基础**：仓库根 **`docker-compose.minio.yml`** 含 `minio` + **`minio-create-bucket`**（自动建桶 **`baodan-attachments`**，并对桶执行 **`mc anonymous set download`**：对象可**匿名下载**，便于内网调试；**不适合生产**。BFF 仍可用 AK/SK 上传/列举。）启动：`docker compose -f docker-compose.minio.yml up -d`；API `http://127.0.0.1:9000`、控制台 `http://127.0.0.1:9001`（`minioadmin` / `minioadmin`）。**存量桶**：可重跑 `minio-create-bucket` 服务或在本机用 `mc` 对桶执行 `anonymous set download`。在 **`frontend/.env.local`** 写入 **`MINIO_*`**（与 `.env.example` 一致）后重启 Next。**MinIO 初始上传**：仓库根 **`node scripts/sync_baodan_to_minio.mjs`**（需本地 **`test/报账单/<单号>/`** 白名单文件）。**链路自检**（读 `frontend/.env.local` 的 `MINIO_*`）：`node scripts/verify_baodan_minio_chain.mjs [报账单号]`。  
**存量行仍含「附件索引」列且需清洗**：`node scripts/clear_baodan_primary_json_from_index.mjs` / `strip_legacy_baodan_attach_index.mjs`；**新库无该列时脚本会跳过**。（依赖 `frontend/node_modules/@aws-sdk/client-s3`；需 `MINIO_ENDPOINT`、`MINIO_ACCESS_KEY`、`MINIO_SECRET_KEY`、`MINIO_BUCKET` 等，见脚本头注释）。脚本扫描 **`test/报账单` 下各报账单子目录**，白名单与 sync 一致，将文件按 **`{MINIO_OBJECT_ROOT}/{报账单号}/{相对路径}`** 写入桶。可先设 **`DRY_RUN=1`** 仅打印将上传的对象键。
6. **主键与幂等**：表主键为自增 `id`；业务唯一键为 **`报账单号`**。导入使用 `INSERT … ON DUPLICATE KEY UPDATE`，同一报账单号重复执行会更新各列（含对象存储三列）。
7. **与 Hermes 联调（附件可读路径）**：以表列 **`附件存储类型` / `附件存储桶` / `附件对象前缀`** 驱动 BFF 从 MinIO 列出并落盘到 **`chat-uploads/<uuid>/`**。**审核技能**：Hermes **禁止**把库表预览中的 Windows 路径当容器内读盘路径。若调用 **`/api/chat/turn`** 时传入 **`context.baodanStageBillNo`**（**支持逗号分隔多单**），BFF 可对**每单**单独拉对象；报账单号个数上限由 **`BAODAN_STAGE_MAX_BILLS`**（默认 5）控制。可选 **`BAODAN_STAGE_FALLBACK_REPO=1`** 在 MinIO 失败时回退本地 `test/报账单`。

## 注册 Windows 服务（管理员 PowerShell）

```powershell
$base = "C:\Program Files\MySQL\MySQL Server 8.4"
$data = "$env:USERPROFILE\mysql-data"
$ini = "$env:USERPROFILE\mysql-test.ini"

@"
[mysqld]
basedir=$base
datadir=$data
port=3307
bind-address=127.0.0.1
mysqlx=0
character-set-server=utf8mb4
collation-server=utf8mb4_general_ci
"@ | Out-File -FilePath $ini -Encoding ascii

& "$base\bin\mysqld.exe" --install MySQL84Test --defaults-file="$ini"
sc.exe config MySQL84Test start= auto
sc.exe start MySQL84Test
```
