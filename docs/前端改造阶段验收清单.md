# 前端改造阶段验收清单

## 通用检查项（每阶段必做）

1. 功能路径是否可达（入口、核心操作、结果反馈）。
2. 异常路径是否可恢复（错误提示、重试、回退）。
3. `npm run lint` 是否通过。
4. `npm run build` 是否通过。
5. 关键接口返回结构是否满足页面渲染需求。

## 阶段1：对话入口可视化

- 普通任务：有执行状态与步骤反馈。
- 追问任务：可承接上下文并展示新执行状态。
- 失败任务：出现错误诊断与恢复动作（重试/跳转修复）。

## 阶段2：任务中心实时化

- 任务列表显示来源、错误摘要、重试次数。
- 任务详情显示步骤日志、输入参数、输出摘要。
- 调度支持创建、启停、状态刷新。

## 阶段3：数据中心可诊断化

- 连通性结果显示 network/auth/queryProbe 三层状态。
- 支持复制诊断信息与快速重试。
- 可从对话失败场景跳转到数据中心处理。

## 阶段4：技能中心产品化

- 技能列表显示版本、来源、状态。
- 详情页支持试运行反馈展示。
- 对话固化后可跳转到技能中心查看。

## 阶段5：系统设置联动可视化

- 模型配置显示同步状态（已同步/待同步/失败）。
- 新任务模型下拉可识别可用性状态。
- 删除模型后新任务页面自动回退“自动”。

## 回归验收

- 回归A（阶段1+2）：对话执行 -> 任务中心追踪 -> 调度动作。
- 回归B（阶段3+4）：连通性失败修复 -> 技能固化 -> 技能试用。
- 最终回归（阶段5）：模型配置 -> 对话执行 -> 数据测试 -> 任务追踪 -> 技能沉淀 -> 调度执行。
